Specifications for my Far Horizons implementation.

== Cluster
The cluster is a sphere containing all of the systems.
For game purposes, the radius is always a whole number.

=== Resources
The richness of resources in the cluster combined with the number of species determines the number of systems in the cluster.

The gamemaster may specify the availability of resources when generating a cluster.
A lower level means fewer resources and more competition for those resources.

The value must be between 0 and 9, with the default being 0.

.Resource Levels
|===
|Level|Systems per Species

|0|6 systems per species
|1|7 systems per species
|2|8 systems per species
|3|9 systems per species
|4|13 systems per species
|5|15 systems per species
|6|20 systems per species
|7|30 systems per species
|8|40 systems per species
|9|50 systems per species
|===

=== Density
Density is measured in number of systems per cubic parsec.
It is a game mechanic that represents the number of systems that contain planets that can be colonized.

The location of the cluster (relative to the core and rim) determines the density.
Moving away from the core decrease the density; moving towards it increases the density.

The default location for clusters is the galactic rim.

.Cluster Location
|===
|Location|Density

|Outer Rim|0.001658 systems per cubic parsec
|Rim|0.002488 systems per cubic parsec
|Inner Rim|0.003732 systems per cubic parsec
|Outer Core|0.005898 systems per cubic parsec

|===

Density influences the pace of the game.
The higher the density, the sooner species will run in to each other.

=== Radius
The number of systems divided by the density determines the total volume of the cluster.
Because the radius must be a whole number, its value will be rounded up.

.Cluster Example
|===
|Number Of Species|Resource Level|Number of Systems|Location|Volume|Radius

|18|0|108|Rim|43,408 pc^3^|22 parsecs
|10|6|200|Outer Core|33,910 pc^3^|21 parsecs
|6|9|300|Outer Rim|180,941 pc^3^|36 parsecs
|===

== Systems
Systems are identified by their co-ordinates.

=== Wormholes
A wormhole allows any ship to move between two systems.
It is bi-directional.
The end-points of the wormhole are fixed when the cluster is created.

Ships do not require FTL engines to pass through a wormhole.

=== Planets
Planets are identified by their system's co-ordinates plus an orbit number.
For example, the planet in the 3^rd^ orbit of system "6 5 4" would be  "6 5 4 #3".

== Species
The number of species determines the size and composition of the cluster.
